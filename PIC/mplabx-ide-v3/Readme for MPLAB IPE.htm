4        What's New in v3.50

·        Reminder: Windows XP operating system is no longer supported  MPLAB X IDE is no longer supported on Windows XP. This means MPLAB X IDE will no longer be tested on Windows XP and JRE8 will be the new default JRE. To continue using Windows XP, JRE 7 will need to be installed. The MPLAB X IDE installer will search for JRE 7 on your system, and if it is not found will recommend sites for download.

·        Reminder: In v3.45, support was added for the 4-wire J-TAG accessory board on the REAL ICE In-Circuit Emulator. For the PIC32 devices that support JTAG programming (specified in the Device Support list) a new checkbox JTAG driver board is available. This feature does not require the user to be in advanced mode.

7        Important Notes

7.1          Switcher Use

If switching from MPLAB IDE v8.xx, you must use the Switcher to switch the PM3 drivers.

If MPLAB IDE v8.xx or earlier is already installed on your computer, you need to run the Switcher program to switch from MPLAB IDE v8 drivers to MPLAB X IDE drivers, which are installed the first time your switch.

The Switcher program is a GUI application called MPDDSwitch (32-bit operating system) or MPDDSwitch64 (64-bit operating system). This should be available as the desktop icon MPLAB Driver Switcher.

For more information on the Switcher program, refer to the MPLAB X IDE online help.

7.2          Checksums

Refer to the MPLAB X IDE readme for any important updates on checksums.

7.3          SST Memory Devices

SST memory devices with large memories (>1MB) will take time for the hex file image to import.  Please allow the time needed for the import to occur.

7.4          Blank Check

In version 2.25 for dsPIC33EV devices, the Blank Check function will check all memory areas including the USER OTP memory.  Future versions of the blank check operation will separate out the USER OTP memory section from all other sections.

7.5          Timestamp

The Output window displays a timestamp for events. For example,

2015-02-19T11:02:41-0700- Completed loading IPE.

This indicates the date (year-month-day), time (hour:minute:second), and the GMT (Greenwich Mean Time) plus or minus offset.

7.6          SQTP Files

SQTP files for Flash Data memory generated prior to MPLAB IPE v2.20 will work with MPLAB PM3 firmware up to v3.00. SQTP file generation for Flash Data memory has been modified to the 32-bit byte order (12345678) and the firmware is updated accordingly. A new SQTP file must be regenerated to work with MPLAB IPE v3.00.

For PIC32 devices only, MPLAB IPE v3.15 provides a dialog box to choose either 32-bit byte order (12345678) or 16-bit byte order (56781234) when loading an SQTP file.

The MPLAB PM3 does not support SQTP for devices with a RAM size of more than 1 MB of memory. Devices affected include PIC32MX2048EFH100 and others that have more than 1MB of memory.

7.7          Saving Split Operating Systems to the PM3 SD Card

In production houses, there may be a need to program several different device family architectures without having access to a computer.

Between MPLAB IPE v3.25  v3.40 the PM3 operating system was split into two parts based on the device architecture (32-bit vs all other devices). Now with MPLAB IPE v3.40, the PM3 operating system and database are split into three parts based on the architectures for 8-, 16- and 32-bit devices.

MPLAB IPE v3.40 includes the feature to save the three PM3 operating system firmwares supporting each of the device family architectures into a PM3 SD card.

When the PM3 environment is selected in Stand-Alone mode, the MPLAB PM3 programmer will load the correct operating system and database from the PM3 SD Card to support the device.

This new feature is available only in the MPLAB IPE v3.40, not in the MPLAB X IDE. The MPLAB X IDE will download one of the split PM3 operating systems based on the architecture of the device chosen in the project when connected to a computer.  To download all three operating systems to the PM3 SD card to allow for complete stand-alone mode, please use the MPLAB IPE v3.40 (or greater).

To save all three operating systems into the PM3 SD card:

  • Insert any supported SD card into PM3 SD card slot.
  • Open MPLAB IPE v3.40 or greater and connect the MPLAB PM3 programmer to the computer.
  • Select Settings >Save firmware into SD Card.
  • On the MPLAB PM3 unit, select a PM3 environment to open. 
  • Once opened, the PM3 will automatically download the correct PM3 operating system split based on the device saved in the PM3 environment.

Points To Be Considered:

  • Please ensure the PM3 operating system firmware version stored on the SD card matches the operating system firmware version on the MPLAB PM3 programmer. A mismatch may occur if an upgrade of the PM3 operating system was performed but the SD card firmware was not updated. Use the MPLAB IPE v3.40 or greater to select the Settings>Save firmware into SD Card option to load the PM3 SD card with the desired PM3 operating system firmware version.
  • Any firmware versions of v3.40 or greater are not compatible with earlier versions of MPLAB IPE. It is highly recommended that the MPLAB PM3 use the firmware packaged within the same MPLAB IPE version.

Prior to MPLAB IPE v3.25

One operating system supported

MPLAB IPE v3.25-3.35

Two operating systems supported (32-bit and all other devices)

MPLAB IPE v3.40 or greater

Three operating systems supported (8-, 16-, 32-bit devices)

 

7.8          Vdd_App and Vdd_Nom

In the power option for PM3, MPLAB 8 uses Vdd_App for Verification and Vdd_Nom is used for all other operation (program, read, erase, blank check). But in MPLAB X IPE, it is the other way around (Vdd_Nom is used for verification and Vdd_App is used for all other operation). Note that in a future release this difference will be removed and MPLAB X IPE will be modified to match with MPLAB v8.
