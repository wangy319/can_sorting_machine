1        Command Line Utility Defined

The Microchip Debugger (MDB) is a command-line interface to Microchip’s hardware and software development tools. As an alternative to using the Microchip MPLAB® X IDE (Integrated Development Environment) graphical interface, the MDB facilitates programming and debugging devices through a Command Prompt interface. The MDB is automatically installed with the MPLAB X IDE.

The MDB is designed for engineers who prefer to use the Command Prompt. The command-line interface to the debuggers is faster and allows more extensive testing to be performed. This is helpful for repetitive tasks or when automating a testing procedure.

The MDB provides a common tool for all Microchip debugging tools (PICkit 3, MPLAB ICD 3, REAL ICE and PM3) support for respective features based on the tool selected.

3        Development Tools Supported

The following development tools work with the MDB:

 

  • MPLAB ICD 3 In-Circuit Debugger
  • PICkit™ 3 In-Circuit Debugger/Programmer
  • MPLAB REAL ICE™ In-Circuit Emulator
  • MPLAB PM3 Device Programmer
  • MPLAB SIM Software Simulator
  • MPLAB Starter Kit on Board (PKOB)
  • Licensed third party debuggers

 

4        Device Support

Device support is dependent on the firmware version installed in the device programmer/debugger. Click the link below to see device support for the debuggers.

Note: UNI/O (11XXXX) devices are supported both through ICSP and socket modules.  All other memory devices are only supported using a socket module. 

           For UNI/O (11XXX) devices, the communication line of the device should be connected to PGD.  VDD/GND of device should be connected to their respective VDD/GND of the ICSP lines.

6        Reference Documents

The MDB command-line utility and supporting documentation are automatically installed with the MPLAB X IDE. There are three ways to access documentation for the MDB:

  1. The MDB User’s Guide may be found may be found in the default location where the MPLAB X IDE was installed:

Program Files>Microchip>MPLABX>vx.xx>docs>MDBUserGuide.pdf

Where vx.xx represents the MPLAB X IDE version.

2.      The MDB Help file provides the most up-to-date information  and is located in the same docs folder:

…>docs>MDB_Help>MDBX_SA.jar

  1. The MDB User’s Guide (PDF) and MDB Help (JAR) can also be accessed by launching the MPLAB X IDE, clicking the Learn & Discover tab, and then clicking on User’s Guide & Release Notes in the Getting Started area. A new window opens showing a list of User’s Guides, Release Notes and Support Documentation where you can find links to the MDB User’s Guide and MDB Help.

On-line help for the other development tools may be found in MPLAB X IDE under Help>Help Contents. The on-line help provides the most up-to-date information.

The following documents may be found on the Microchip website:

·        MPLAB ICD 3 In-Circuit Debugger User's Guide for MPLAB X IDE

·        MPLAB PM3 User's Guide for MPLAB X IDE

·        PICkit 3 Programmer/Debugger User's Guide for MPLAB X IDE

·        MPLAB REAL ICE In-Circuit Emulator User's Guide for MPLAB X IDE

·        Development Tools Design Advisory (DS51764)

·        Programming Specifications for devices (various DS numbers)

7        What's New in v3.50

·        Windows XP operating system is no longer supported – MPLAB X IDE is no longer supported on Windows XP. This means MPLAB X IDE will no longer be tested on Windows XP and JRE8 will be the new default JRE. To continue using Windows XP, JRE 7 will need to be installed. The MPLAB X IDE installer will search for JRE 7 on your system, and if it is not found will recommend sites for download.

8        Repairs and Enhancements in v3.50

MDB-35

There is no access to read the PIC32 peripheral memory regions from within the MDB command line. If MDB executes commands from a file and a "Dump" command is used, MDB will fail.
