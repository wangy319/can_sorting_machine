1        Command Line Utility Defined

MPLAB IPE Command Line Interface (IPECMD) is a 32/64-bit command-line interface to the MPLAB X IDE. This interface is designed for programming devices in a production environment.

The IPECMD provides a common tool for the Microchip programming tools (PICkit 3, MPLAB ICD 3, MPLAB REAL ICE and MPLAB PM3). Support for the respective features is based on the programming tool selected.

Possible uses for this interface include:

·        Setting up an MPLAB PM3 for stand-alone programming or a PICkit 3 for Programmer-To-Go.

·        Cross-platform programming.

·        Programming devices without using an IDE.

·        SQTP programming.

·        Environment support.

13    Important Notes

13.1         Java Requirements

The IPECMD.jar is written in Java. Therefore, you will need the JRE (Java Runtime Environment) to run your script. In order to avoid conflict, please use the JRE that is distributed with MPLAB X IPE installation. You can set the environment path using the MPLAB X IPE Installation directory “..\sys\java\jrex.xx\java-windows\bin” for Windows and “..\sys\java\jrex.xx\bin” for Linux/Mac.

13.2         Specifying Options

Each option must be immediately preceded by a switch, which can be either a dash “-’ or a slash “/” and options must be separated by a single space. Example:

>java -jar ipecmd.jar  /P16F877 /Fc:\mycode.hex /M
or
>java -jar ipecmd.jar   -P16F877 -Fc:\mycode.hex -M

Notes:

·        For Windows, commands and their parameters are not case-sensitive. For Linux/Mac, the file path must be case-sensitive.

·        Commands will be processed according to command order of precedence not the order in which they appear on the command line.

·        The program will return an exit code upon completion which will indicate either successful completion or describe the reason for failure.

13.3         Programming Speed Improvement (Applicable only for MPLAB PM3)

To improve speed when programming your device, after the first time it has been programmed, remove the /F option from the command. Removing the file specification /f allows the tool to reuse the image contained within the tool’s memory.

14.1         Using the Short Name

To select the tool using the short name, use -TP to select programming tool. For example:

-TPPM3 to select the MPLAB PM3

-TPPK3 to select the PICKIT 3

-TPRICE to select the MPLAB REAL ICE

-TPICD3 to select the MPLAB ICD 3

To select the tool using the serial number, use -TS to select programming tool. For example:

-TSBUR1234567 to select the tool with the serial number BUR1234567

Note: The MPLAB PM3 does not carry a unique identifier, therefore, the -TS switch is not applicable.

14.5         How to Operate IPECMD with Multiple Instances

By default, you can work with up to five (5) instances of IPECMD. If you want to have more instances, you will need to modify the “mchpdefport” file manually.

14.5.1    Use and Format of “mchpdefport” File

The “mchpdefport” file provides the information necessary for tool hot-plug use to both the IPECMD and to the low-level USB library (DLL, so, or dylib file).

The format for this file is as follows:

localhost

30000

30002

30004

30006

30008

 

The first line indicates the host name on which the IPECMD is running.

The other lines represent port or socket numbers through which the low-level library communicates with the upper-level IPECMD. Each instance of the IPECMD will be assigned to a different port or socket. All communications between the instances of the IPECMD should be hidden from the user.

For up to five (5) instances of IPECMD, you do not need to alter this file. If you want more than five instances, you can edit the file to add more port or socket numbers.

14.5.2    Location of The “mchpdefport” File

Within a default installation of the IPE, the “mchpdefport” file can be found in the following place, depending on the operating system:

16    Frequently Asked Questions

NOTE: IPECMD can be invoked using java –jar on any Windows, Mac, or Linux machine:

>java -jar ipecmd.jar -TPPK3 -P18F4550 -M –F”c:/demo.hex”

Or, if using a Windows-based operating system:

>ipecmd.exe -TPPK3 -P18F4550 -M –F”c:/demo.hex”

16.1         How to program a device using the PICkit 3?

> java -jar ipecmd.jar  -TPPK3 -P18F4550 -M –F”c:/demo.hex”

or

> java -jar ipecmd.jar  -TPPK3 -P18F4550 -M –F”/home/project/demo.hex”

16.2         What happens if more than one PICkit 3 is connected?

The IPECMD will prompt for tool selection, for example:

Available Tool List

1 ICD 3   S.No: JIT102832586

2 PICkit 3 S.No: BUR112281031

3 PICkit 3 S.No: BUR120299953

More than one PK3 is connected...

Please enter your choice: _

If you do not specify which tool to the IPECMD should connect to, the IPECMD will prompt you for the tool selection. Otherwise, you can specify the tool by using the –TS key (i.e., –TSBUR10234534).

16.3         How to program the target with different tools?

To program using MPLAB PM3

> java -jar ipecmd.jar  -P24FJ256GB106 –TPPM3 -M –F”e:/pic24.hex”

To program using PICkit 3:

> java -jar ipecmd.jar  -P24FJ256GB106 -TPPK3 -M –F”e:/pic24.hex”

To program using MPLAB REAL ICE:

> java -jar ipecmd.jar  -P24FJ256GB106 -TPRICE -M –F”e:/pic24.hex”

To program using ICD 3:

> java -jar ipecmd.jar  -P24FJ256GB106 -TPICD3 -M –F”e:/pic24.hex”

16.4         How to read the target with PICkit 3?

Read using PK3 and export the file:

> java -jar ipecmd.jar  -P24FJ256GB106 -TPPK3 –GF”e:/pic24.hex”

16.5         How to erase the target?

Erase using PK3:

> java -jar ipecmd.jar  -P24FJ256GB106 -TPPK3 -E

16.6         How to verify the target?

Verify using PK3:

> java -jar ipecmd.jar  -P24FJ256GB106 -TPPK3 -Y

16.14     What is Batch Mode?

Batch Mode operation will initialize the tool once and perform the same operations repeatedly. To repeat the operation, the user will press the space key. Batch Mode will exit if there is a failure of operation or the user requests to stop.

 

Warning: Batch Mode is suggested to use only with direct invocation of tool because it will enter into a wait state for user input.

 

Example:

> java -jar ipecmd.jar   -TPPM3  -P18F4580  -FHEXCODE.HEX  -M  -OB

PM3 Program Report

24-Jan-2014, 15:49:35

Device Type: PIC18F4580

Program Succeeded

Batch Mode Count: 1

Batch Mode: Press Enter to repeat the operation.

Press Enter to program repeatedly or press Ctrl+C to abort the operation.

16.15     How to repeatedly program the target once the environment is loaded?

Batch Mode can be used in order to program the environment settings repeatedly.

> java -jar ipecmd.jar  –tppm3 –blc”E:/myenv” –M –OB

16.16     Is COM Port support available for MPLAB PM3 in IPECMD?

Communication Ports 1 through 255 supported. Com port 5 indicates USB and other Com ports indicate RS232.

16.17     How to set MPLAB PM3 on Com Port 2?

>java -jar ipecmd.jar  –p18F4580 -2 –f”E:/PIC18F4580.hex” –M –Y

MPLAB PM3 on COM 2 is set to PIC18F4580. The file PIC18F4580.hex is sent to the MPLAB PM3. Voltages are set to the device’s default values and the device is programmed. After programming, the device is verified against the contents of PIC8F4580.hex.

16.18     How to set MPLAB PM3 on Com Port 3?

>java -jar ipecmd.jar  –p18F4580 -3 –f”E:/PIC18.hex” –M –Z

MPLAB PM3 on COM 3 is set to PIC18F4580. The file PIC18.hex is sent to the MPLAB PM3. Voltages are set to the device’s default values. All of the device’s memory is programmed with blank values in the location not specified by PIC18.hex. The –Z switch specifies PM3 to preserve EEData.

16.19     How to generate an SQTP File?

To generate an SQTP file, use MPLAB IPE, select 'Advanced', and open the SQTP tab.

16.20     What is an Environment?

Environments allow you to save specific tool settings so that all of the same settings can be reloaded again in a different programming session. Environments are supported under all tools as either .pen files or .pm3 files. An environment can be created using the –B switch.

17    Command Line Examples

17.1         Displaying Help

>java -jar ipecmd.jar  /?  or >java -jar ipecmd.jar  -?

17.2         Program with SQTP Num File

> java -jar ipecmd.jar  /P12F508 /fvolts.hex /ssqtp.num /M /TPPM3

MPLAB PM3 is selected and set to PIC12F508. Hex file VOLTS.HEX is sent to the MPLAB PM3. Voltages are set to the device's default values and one entry from the SQTP file SQTP.NUM is sent. All of the device's memory is programmed with blank values in the locations not specified by VOLTS.HEX and SQTP.NUM. If programming is successful, the SQTP entry is marked as used.

17.3         Program the Target with a Hex File

> java -jar ipecmd.jar  /P16F1938 /FAPPCODE.HEX /M /TPPK3

PICKIT 3 is selected and set to a PIC16F1938. The file APPCODE.HEX is sent to the PICKIT 3. Voltages are set to the device's default values. All of the device's memory is erased and programmed with blank values in the locations not specified by APPCODE.HEX.

17.4         Program the Target without a Hex File

> java -jar ipecmd.jar  /P16F1938 /M /TPPM3

MPLAB PM3 is set to a PIC16F1938. The device is erased and programmed with the current contents of the device programmer at the device’s default voltage values.

17.5         Verify a Hex File Against a Target

> java -jar ipecmd.jar  /P16F887 /FHEXCODE.HEX /Y /TPPM3

MPLAB PM3 is set to a PIC16F887. Voltages are set to the device's default values. The device is verified against the contents of HEXCODE.HEX.

17.6         Verify Hex File Against a Target after Programming

> java -jar ipecmd.jar  / P16F887 /FHEXCODE.HEX /M /Y /TPPM3

MPLAB PM3 is set to a PIC16F887. The file HEXCODE.HEX is sent to the MPLAB PM3. Voltages are set to the device's default values and the device is erased and programmed. After programming, the device is verified against the contents of HEXCODE.HEX.

Note: The Verify with (/M) operation implicitly performs a Verify when it completes the programming portion of the operation. Specifying a Verify (/Y) in the above manner effectively causes two Verify operations to be performed.

17.7         How to Preserve EEData

> java -jar ipecmd.jar  /P18F4220 /FAPPCODE.HEX /Z /M /TPPM3

MPLAB PM3 is set to a P18F4220. The file APPCODE.HEX is sent to the MPLAB PM3. Voltages are set to the device's default values. All of the device's memory is erased and programmed, with blank values in the locations not specified by APPCODE.HEX. The /Z switch specifies PRO MATE to preserve EEData.

17.8         How to Preserve EEdata After Programming With Blank Values

> java -jar ipecmd.jar  /P18F4220 /FAPPCODE.HEX /E /Z /M -TPPM3

MPLAB PM3 is set to a P18F4220. The file APPCODE.HEX is sent to the MPLAB PM3. Voltages are set to the device's default values. All of the device's memory is erased and programmed, with blank values in the locations not specified by APPCODE.HEX.

Note: The Erase Flash device (/E) Erases EEData, even though /Z switch specifies MPLAB PM3 to preserve EEData, because /E switch overrides /Z switch.

17.39     How to Use PICKIT 3 in (PTG) Programmer-to-Go Mode

Example-1: Consider a target is externally powered and the image is loaded into the PICKit 3 as image1.

>java -jar ipecmd.jar  –TPPK3 –p18F4580 –OG”image1” –f”E:/p18F4580.hex”

The following settings are applied while executing the above command:

·        Target has its own power

·        PICKIT 3 will auto calculate ranges to be programmed

·        Program memory will be erased prior to programming

 

Example-2: Consider a target is externally powered and the image is loaded into the PICkit 3 as image1 and the program memory should not be erased before programming.

>java -jar ipecmd.jar  –TPPK3 –p18F4580 –OG”image1” –f”E:/p18F4580.hex” –OH

The following settings are applied while executing the above command:

·        Target has its own power

·        PICKIT 3 will auto calculate ranges to be programmed

·        Program memory will NOT be erased prior to programming

 

Example-3: Consider a target is externally powered and the image is loaded into the PICkit 3 as image1 and select only the program memory region.

>java -jar ipecmd.jar  –TPPK3 –p18F4580 –OG”image1” –f”E:/p18F4580.hex” –OCP

The following settings are applied while executing the above command:

·        Target has its own power

·        Selected memory areas will be programmed (Program memory is selected in the above command)

·        Program memory will be erased prior to programming

 

Example-4: Consider Target is powered by Pickit-3 and load the image into PK3 as image1

>java -jar ipecmd.jar  –TPPK3 –p18F4580 –OG”image1” –f”E:/p18F4580.hex” –W

The following settings are applied while executing the above command:

·        PICKIT 3 powers the target

·        PICKIT 3 will auto calculate ranges to be programmed

·        Program memory will be erased prior to programming

 

Note: During the programming operation the PICkit 3 “Busy” LED will remain lit continuously. The “Target” LED will light if powering from PICkit 3, but will remain off if the target has its own power supply.

19    IPECMDBoost

MPLAB IPECMDBoost is a 32/64-bit command-line interface to the MPLAB X IDE. This interface is designed for programming devices in a production environment with improved speed.

The IPECMDBoost provides a common tool for the Microchip programming tools (PICkit 3, MPLAB ICD 3, MPLAB REAL ICE and MPLAB PM3). Support for the respective features is based on the programming tool selected.

19.1          Using IPECMDBoost

IPECMDBoost is available under the Installed_Location/MPLABX/v3.x/mplab_ipe/ ipecmdboost/ folder.

Copy the ipecmdboost.exe to the …/MPLABX/v3.x/mplab_ipe/ folder.

·        Follow the same commands as ipecmd.

19.2         Limitations

·        IPECMDBoost will be always be running in the background, therefore do not use multiple instances of IPECMDBoost.

·        Running MPLAB IPE or MPLAB X IDE at the same time that IPECMDBoost is enabled may cause these tools to have connection issues.

·        IPECMDBoost uses port number 2012 and 2013, please avoid these port numbers in your applications.

·        This supports only the Windows platform.

19.3         Tips and Tricks

·        If there are any connection issues with the tools, please ensure the firewall is not blocking the port numbers 2012 and 2013 which are used by the IPECMDBoost utility.

·        If you have problems using any programming tools with MPLAB X IPE or MPLAB X IDE after using the IPECMDBoost utility, please go to the Windows Task Manager and end the java.exe process and try again.

20    Troubleshooting

20.1         Failed to communicate to MPLAB REAL ICE /MPLAB ICD 3/PICKit 3

·        Make sure only one session is communicating with the MPLAB REAL ICE, MPLAB ICD 3 or PICKIT 3. If the MPLAB REAL ICE, MPLAB ICD 3 or PICKIT 3 is already loaded in the MPLAB IPE, it will fail to communicate.

·        Ensure the driver is switched to MPLAB X. Drivers do not have to be switched for PICKIT 3.

·        Make sure the compatible Firmware type is loaded on the MPLAB REAL ICE, MPLAB ICD 3 or PICKIT 3 for the selected device.
To do that open MPLAB IPE, select the specific device and select the MPLAB REAL ICE, MPLAB ICD 3 or PICKIT 3. MPLAB IPE will detect the firmware and, if needed, it will update the firmware. (By default the MPLAB REAL ICE, MPLAB ICD 3 or PICKIT 3 will automatically update the firmware).

20.2         Programming Failed

·        Make sure the selected Hex file is compatible with the current device.

·        Check the selected Memory Region is available in the device.

·        Check the selected device requires erase before programming.

21    Notes

  1. If downloading firmware, only the communications port and the /d option can be specified.
  2. If not downloading firmware, a device must be specified.
  3. If a Hex file is specified, only the locations specified in the hex file are programmed. If an SQTP file is also specified, then the entire device is programmed.
  4. If a hex file is not specified, the entire contents of the selected device programmer are programmed into the device.
  5. Any values not specified in the hex file are set to blank (erased) values.
  6. SQTP lines are marked as used in SQTP file only if programming is successful.
  7. The MPLAB REAL ICE, MPLAB ICD 3 or PICKIT 3 will automatically download the latest firmware if required, this will cause delay to the operation.
  8. Administrator privileges are required to perform environment operations and to export files to systems drive on Windows 7 and Windows 8.
  9. Calibration memory value will be overwritten only if calibration memory is present within program memory region.
